AI-Artist OpenClaw Hosted LLM Flow

The selected architecture uses OpenClaw as the agent control plane, hosted OpenAI models for reasoning, ComfyUI for image generation, and a FastAPI/OPA safety layer for reuse, audit, and privileged execution.

Core Flow

External Request
 -> OpenClaw Gateway
 -> AI-Artist Main Agent
 -> FastAPI Safety Service
 -> Request Canonicalizer
 -> Policy Context Builder
 -> OPA Policy Layer
 -> Request Classifier
    -> repeat read-only query
       -> Reuse Gate
       -> Source Freshness Check
       -> Approved Response Cache
       -> Final Validation
       -> Final Response
    -> fresh query or action request
       -> OpenClaw Main Agent
       -> OpenClaw Agent Runtime
       -> Restricted OpenClaw Sub-Agents
       -> Validation / Compare / Retry / Synthesis
       -> Final Validation
       -> Execution Policy Gate
       -> Output Tool Agent
       -> Channels / Platforms

Control Rules

  • OPA is checked before reuse and before privileged execution.
  • Hosted OpenAI models are the primary LLM backend.
  • OpenClaw owns agent workspaces, channel adapters, and tool hooks.
  • Every implementation task has a validation test before it can be marked done.
  • Only approved read-only responses are eligible for replay.
  • Source freshness must be checked before cache reuse.
  • Secrets stay out of chat memory, agent memory, prompts, logs, and audit payloads.
  • Output Tool Agent owns external delivery and action execution.
